# Upgrade Cadence

Temenos Transact release adoption plan.

## Release Types

| Type | Frequency | Adoption Target | Notes |
|---|---|---|---|
| Major Release | Annual | Within 12 months | Full regression required |
| Service Pack | Quarterly | Within 3 months | Targeted testing |
| Patch | Monthly | Within 1 month | Critical fixes only |
| Hotfix | As needed | Within 1 week | Emergency only |

## Adoption Process

| Step | Activity | Owner | Duration |
|---|---|---|---|
| 1. Release Notes | Review release notes and known issues | L3 | 2 days |
| 2. Impact Assessment | Assess impact on customisations and integrations | L3 | 5 days |
| 3. Test Plan | Define test scope for the upgrade | L2/L3 | 3 days |
| 4. Non-Prod Deploy | Deploy to SIT, run tests | L3 | 5 days |
| 5. UAT | Business validation | Business | 5 days |
| 6. Prod Deploy | Deploy to production | L3 | 1 day (change window) |
| 7. Post-Deploy | Monitor, verify | L1/L2 | 2 days |
