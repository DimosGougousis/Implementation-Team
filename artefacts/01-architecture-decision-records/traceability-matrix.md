# Traceability Matrix

Maps requirements to ADRs to implementation artefacts.

| Requirement | ADR | Implementation Artefact | Status |
|---|---|---|---|
| | | | |

## How to Use
1. Each business requirement that drives an architectural decision gets a row
2. Link to the ADR that captures the decision
3. Link to the implementation artefact (code repo, config, infra)
4. Track status: Planned → In Progress → Implemented → Verified
