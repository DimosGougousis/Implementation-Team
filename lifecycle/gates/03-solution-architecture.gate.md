# Stage 3: Solution Architecture

## Purpose
Author Architecture Decision Records for all significant decisions, finalise the target-state architecture, create the integration dependency map, define the test strategy framework, and begin the post-go-live operating model design.

## Entry Criteria
- [ ] Fit-Gap gate passed
- [ ] All gaps documented with proposed resolutions
- [ ] Integration landscape identified
- [ ] Architecture review board convened

## Key Activities
1. Author ADRs for all significant architectural decisions
2. Finalise target-state reference architecture
3. Design integration architecture (APIs, events, adapters)
4. Define integration contracts and SLA matrix
5. Establish test strategy framework and entry/exit criteria
6. Begin post-go-live operating model design
7. Update fit-gap register with architecture decisions

## Artefacts Produced / Updated

| Artefact | Action | What Changes |
|---|---|---|
| Architecture Decision Records | Created | ADR register, initial ADRs, traceability matrix |
| Integration Dependency Map | Created | Register, dependency graph, adapter strategy, SLA matrix |
| Test Strategy & KPI Dashboard | Framework | Strategy document, entry/exit criteria |
| Post-Go-Live Operating Model | Started | Initial target operating model |
| Target-State Reference Architecture | Updated | Finalised with integration and security details |
| Fit-Gap Register | Updated | Linked to ADRs |

## Exit Criteria (Gate)
- [ ] All significant decisions documented as ADRs
- [ ] Target-state architecture approved
- [ ] Integration dependency map complete
- [ ] API and event contracts defined
- [ ] Test strategy framework approved
- [ ] Operating model initial design complete
- [ ] Steering Committee approved architecture

## Steering Committee Checkpoint
Present: architecture decisions, integration complexity, test approach, operating model design, risk assessment.
