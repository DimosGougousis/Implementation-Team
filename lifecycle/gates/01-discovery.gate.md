# Stage 1: Discovery

## Purpose
Understand the bank's current state, strategic objectives, regulatory context, and programme scope. Establish the initial target-state reference architecture and confirm the implementation approach.

## Entry Criteria
- [ ] Programme charter approved
- [ ] Key stakeholders identified and engaged
- [ ] Access to current-state documentation granted

## Key Activities
1. Conduct stakeholder interviews across business and technology
2. Document current-state architecture and pain points
3. Define strategic objectives and success criteria
4. Establish initial target-state reference architecture
5. Confirm programme scope, timeline, and budget
6. Identify key risks and dependencies

## Artefacts Produced / Updated

| Artefact | Action | What Changes |
|---|---|---|
| Target-State Reference Architecture | Created | Initial business capability map, component model, technology standards |

## Exit Criteria (Gate)
- [ ] Current-state documented
- [ ] Strategic objectives agreed
- [ ] Initial target-state architecture produced
- [ ] Programme scope confirmed
- [ ] Key risks identified
- [ ] Steering Committee briefed

## Steering Committee Checkpoint
Present: programme scope, strategic alignment, initial target-state, key risks, resource requirements.

## Your Contribution at This Stage
- Lead stakeholder discovery workshops
- Document current-state architecture
- Produce initial target-state reference architecture
- Identify integration complexity and data migration scope
- CV competencies: Systems Thinking, Stakeholder Orchestration
