# Stage 2: Fit-Gap & Process Design

## Purpose
Systematically assess each Temenos Transact module against business requirements, score gaps for customisation vs workaround, establish the customisation budget, and begin data quality profiling.

## Entry Criteria
- [ ] Discovery gate passed
- [ ] Business requirements documented
- [ ] Transact environment available for assessment
- [ ] Fit-gap workshop schedule agreed

## Key Activities
1. Conduct fit-gap workshops per Transact module
2. Score each requirement (Fit / Partial / Gap)
3. Evaluate customisation vs workaround for each gap
4. Establish and track customisation budget
5. Begin data quality profiling across source systems
6. Document business process redesign requirements

## Artefacts Produced / Updated

| Artefact | Action | What Changes |
|---|---|---|
| Fit-Gap Register | Created | Master register with all module assessments |
| Data Quality Scorecard | Started | Initial profiling results |
| Target-State Reference Architecture | Updated | Refined based on fit-gap findings |

## Exit Criteria (Gate)
- [ ] All Transact modules assessed
- [ ] Fit-gap register complete with scores
- [ ] Customisation budget established
- [ ] Data profiling completed for all domains
- [ ] Business process redesign documented
- [ ] Steering Committee briefed on customisation scope

## Steering Committee Checkpoint
Present: fit-gap summary, customisation budget, key gaps, data quality baseline, recommended approach.

## Your Contribution at This Stage
- Facilitate fit-gap workshops with business stakeholders
- Score requirements and evaluate customisation trade-offs
- Establish customisation budget and tracking
- Begin data quality assessment
- CV competencies: Fit-Gap Analysis, Business Process Alignment
