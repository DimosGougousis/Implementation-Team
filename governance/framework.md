# Governance Framework

Adapted for Temenos Transact implementation programmes.

## Principles

1. **Stage-gated progression** — No programme advances without passing the gate review for the current stage.
2. **Evidence-based gates** — Gate criteria are checklist items backed by artefacts, not opinions.
3. **Contribution visibility** — Every stage must log personal contribution and value to the bank.
4. **Customisation governance** — All customisations require ADR justification and budget tracking.
5. **Artefact-first** — Deliverables are concrete files, not slide decks or verbal updates.

## Gate Review Process

1. Programme lead completes all exit criteria for the current stage
2. Artefacts are reviewed against the gate template
3. Gate decision: Pass / Conditional Pass / Fail
4. Decision recorded in `STAGE-STATUS.md` gate history
5. Steering Committee checkpoint (if applicable)

## Roles

| Role | Responsibility |
|---|---|
| Programme Lead | Owns stage progression, gate readiness |
| Design Authority | Reviews ADRs, architecture conformance |
| Implementation Consultant | Delivers artefacts, logs contribution |
| Steering Committee | Approves gate passage at key stages |
